# 产品使用手册

本文面向评委、老师和最终使用者，说明如何打开系统、如何进行文字/语音/图片问答，以及如何进入后台管理。

## 1. 访问方式

部署完成后，默认通过浏览器访问前端页面。

如果是本机开发环境：

- 游客端：`http://127.0.0.1:5173/`
- 管理后台：`http://127.0.0.1:5173/admin.html`

如果是测试服务器原生部署并通过 SSH 隧道访问：

- 游客端：`http://127.0.0.1:18080/`
- 管理后台：`http://127.0.0.1:18080/admin.html`

## 2. 游客端使用

### 2.1 文本问答

在输入框中直接输入问题，然后点击发送或按回车即可。

示例：

- “开放时间是什么时候？”
- “请推荐一条适合亲子游的路线”
- “这个景点有什么历史背景？”

### 2.2 语音问答

点击麦克风按钮开始录音，说完后再次点击结束录音，系统会自动识别语音并发起问答。

使用建议：

- 尽量在相对安静的环境下录音
- 说话尽量连续，避免过长停顿
- 如果识别结果不准确，可以改用文本输入

### 2.3 图片问答

点击输入区的 `+` 号上传图片，或直接把图片拖入对话框。

上传后可以继续输入问题，也可以不输入文字直接发送。系统会自动补默认问句，例如：

- `请帮我看看这张图片。`

常见提问方式：

- “这是哪个景点？”
- “这张图里是什么建筑？”
- “帮我介绍一下这张图片里的地方”

### 2.4 停止生成

如果当前回答不需要了，可以点击 `停止生成`。

这会中止当前一次问答和语音播报，方便你立即切换到下一轮提问。

## 3. 管理后台

点击右上角或直接进入 `admin.html` 可打开管理后台。

后台主要用于：

- 配置数字人形象和音色
- 管理知识库资料
- 查看会话记录
- 查看数据统计和报告
- 预览当前数字人效果

默认后台账号由部署环境配置决定，通常为 `admin`，密码需要在部署时单独设置。

## 4. 推荐操作流程

1. 先打开游客端确认前端正常加载
2. 输入一个简单问题，确认文字问答可用
3. 再测试语音问答，确认识别和播报正常
4. 再上传一张图片，确认图文问答可用
5. 最后进入后台检查数字人配置和会话记录

## 5. 常见问题

### 5.1 页面空白或数字人不显示

先检查浏览器是否能打开前端静态资源和 `live2d` 资源。

如果是服务器部署，优先确认：

- 前端是否已构建完成
- `nginx` 是否已启动
- `http://服务器地址/live2d/...` 是否能直接访问

### 5.2 图片上传失败

常见原因：

- 图片超过大小限制
- 图片格式不支持
- 后端上传目录权限不对
- 后端服务未启动

### 5.3 语音没有回复

请检查：

- 麦克风是否授权
- 后端 `ASR` 是否启动正常
- 浏览器控制台是否有报错

### 5.4 回答内容像“兜底话术”

这通常表示知识库检索结果不够准确，或者模型没有拿到足够上下文。可以：

- 换一个更具体的问题
- 检查知识库是否已导入
- 查看后台会话和知识库配置

## 6. 运维提示

如果你在测试服务器上部署，建议同时保留以下检查入口：

- 后端健康检查：`/api/v1/health`
- 后端日志：`/opt/ai-chat-live2d/data/logs/backend.log`
- 前端/反代日志：`/opt/ai-chat-live2d/data/logs/nginx-error.log`

这样出现异常时，可以快速判断是前端、后端还是模型资源问题。
