# 数字人体验调节与后台情绪预览设计

## 背景

当前游客端已经具备 Live2D 加载、口型同步、情绪参数映射、会话阶段切换和情感熔岩灯。后台“数字人管理”也能预览 Live2D 模型并配置音色、Prompt、TTS 情绪开关。

本次优化聚焦数字人体验：

- 游客端数字人框需要自由缩放，避免不同屏幕和不同 Live2D 模型身位不合适。
- Live2D 模型本身需要可调缩放和 X/Y 偏移，解决“框变大但人物仍然站位不对”的问题。
- 后台数字人管理需要新增“情绪与表情”栏，方便查看每种情绪对应的 Live2D 表情、参数和熔岩灯状态。
- 游客端已有情感熔岩灯，后台应复用同一套视觉规则，保证调试时看到的状态和游客端一致。

## 目标

采用“后台默认展示参数 + 游客端本机临时覆盖”的混合方案。

- 后台保存数字人档案级默认值：模型缩放、模型水平偏移、模型垂直偏移、默认舞台高度。
- 游客端读取后台默认值后，允许用户临时拖拽缩放舞台、调整模型缩放和偏移，并用 `localStorage` 记住本机偏好。
- 后台数字人页新增情绪预览面板，支持 `neutral / happy / thinking / excited / sad` 五种情绪切换，Live2D 预览和熔岩灯同步变化。
- 不改变现有 WebSocket 协议，不影响 TTS、RAG、ASR 主链路。

## 非目标

- 本轮不做复杂拖拽布局系统，不引入第三方 resize 库。
- 本轮不做多套表情动作编排器，只提供当前五种情绪的预览和参数说明。
- 本轮不做服务端实时情绪监控推送，后台情绪预览为配置调试工具，不连接游客端实时会话。
- 本轮不重写 Live2D 资源解析逻辑，不要求每个模型都具备同名表达式文件；缺失时继续使用参数 overlay fallback。

## 方案选择

### 方案 A：仅前端本地保存

实现最快，只在游客端 `localStorage` 保存舞台尺寸和模型参数。缺点是管理员无法统一配置默认展示效果，不适合多个数字人档案。

### 方案 B：仅后台统一保存

配置可控，演示一致性好。缺点是游客端无法根据自己的屏幕临时调整，不够灵活。

### 方案 C：后台默认值 + 游客端本机覆盖

推荐采用。后台负责“这个数字人默认应该怎么站”，游客端负责“这台设备现在怎么舒服地看”。这样既能支持管理员调试，又不会牺牲现场演示的灵活性。

## 数据模型

在 `avatar_config` 增加展示字段：

- `display_scale`: Live2D 模型默认缩放，建议范围 `0.60` 到 `1.80`，默认 `1.00`。
- `display_offset_x`: Live2D 模型默认水平偏移，单位为舞台宽度比例，建议范围 `-0.50` 到 `0.50`，默认 `0`。
- `display_offset_y`: Live2D 模型默认垂直偏移，单位为舞台高度比例，建议范围 `-0.50` 到 `0.50`，默认 `0`。
- `stage_height`: 游客端默认舞台高度，单位 `px`，建议范围 `320` 到 `760`，默认沿用当前视觉高度。

后端保持轻量 SQLite 兼容迁移策略：启动时检查缺列并补齐默认值。API 层在 `GET /admin/avatar/config`、`PUT /admin/avatar/config`、游客端头像列表中返回这些字段。

## 前端组件设计

### Live2DStage

`Live2DStage` 增加可选 props：

- `modelScale?: number`
- `modelOffsetX?: number`
- `modelOffsetY?: number`
- `presentation?: AvatarPresentation`

`resizeModel()` 根据舞台尺寸、模型原始 bounds 和展示参数计算最终 scale 与 position。默认值保持当前行为，避免影响现有页面。

### AvatarDisplayControls

新增轻量控制面板，用于游客端和后台复用：

- 模型缩放滑杆。
- 水平偏移滑杆。
- 垂直偏移滑杆。
- 舞台高度滑杆或输入。
- 重置按钮。

游客端修改后写入 `localStorage`，key 包含 `avatarProfileId`，避免不同数字人互相污染。后台修改后进入表单，保存到数据库。

### EmotionLamp

将游客端现有熔岩灯 markup 抽成可复用组件，继续使用 `buildEmotionLampStyle()` 和 `EMOTION_VISUALS`。

后台和游客端共享：

- 情绪标签。
- 阶段标签。
- 置信度。
- 来源。
- 关键词和原因。

### AdminEmotionPreviewPanel

后台数字人管理新增“情绪与表情”栏：

- 五个情绪按钮：中性、愉快、思考、兴奋、克制。
- 预览阶段切换：`preview` / `final`。
- Live2D 预览实时调用 `setAvatarPresentation()`。
- 显示当前情绪的参数说明和表达式映射名称。
- 显示与游客端一致的情感熔岩灯。

## 游客端交互

游客端舞台提供一个不打扰主体验的“显示调节”入口：

- 默认折叠为小按钮，放在舞台卡片右上角或 caption 区域。
- 展开后显示模型缩放、X/Y 偏移、舞台高度和重置。
- 舞台高度改变时只影响数字人区域，不重排聊天输入底部固定逻辑。
- 模型参数改变时即时作用于 Live2D，不需要刷新。

保存策略：

- 后台默认值来自当前激活数字人档案。
- 本机覆盖值存在时优先生效。
- 点击“重置”清除本机覆盖并恢复后台默认值。

## 后台交互

后台数字人管理页拆成三块更清晰的区域：

- 实时预览：Live2D 当前形态、模型路径、音色、TTS 语速。
- 展示参数：缩放、偏移、舞台高度，保存后成为游客端默认值。
- 情绪与表情：情绪按钮、阶段切换、熔岩灯、参数说明。

后台情绪预览只影响当前管理页预览，不会把预览情绪写入数据库，也不会影响游客端正在进行的会话。

## 数据流

1. 后端启动时迁移 `avatar_config` 展示字段。
2. 管理员打开后台，读取当前数字人档案配置。
3. 管理员调整展示参数，Live2D 预览即时变化。
4. 管理员保存配置，后端校验范围并写入数据库。
5. 游客端加载当前激活数字人档案，得到默认展示参数。
6. 游客端读取本机覆盖，如果存在则覆盖后台默认值。
7. 游客端用户调整显示参数后写入 `localStorage`，Live2D 即时更新。

## 校验与回退

- 后端校验展示字段范围，越界返回 422。
- 前端输入也做范围限制，但以后端为准。
- Live2D 模型加载失败时，展示控制面板仍可显示，但控件禁用或提示“模型未加载”。
- localStorage 解析失败时忽略本机覆盖，使用后台默认值。
- 某个模型缺少对应 expression 文件时，继续使用现有参数映射 fallback。

## 测试计划

### 前端单元测试

- 展示参数合并：后台默认值、本机覆盖、重置逻辑。
- 情绪灯组件：不同情绪和阶段能生成稳定样式变量。
- 情绪预览状态：点击情绪按钮后 `presentation` 更新为正确情绪。

### 后端测试

- 旧 schema 启动迁移后补齐展示字段。
- `GET /admin/avatar/config` 返回展示字段。
- `PUT /admin/avatar/config` 能保存合法展示字段。
- 越界展示字段被拒绝。

### 构建与人工验收

- `npm run build` 通过。
- 游客端可以调整舞台高度、模型缩放和 X/Y 偏移，刷新后本机偏好仍保留。
- 后台保存默认展示参数后，新的游客端设备能看到默认效果。
- 后台点击五种情绪时，Live2D 预览和熔岩灯同步变化。
- 现有聊天、TTS、口型同步和会话切换不回归。

## 验收标准

- 数字人舞台不再只能固定尺寸，能适配不同 Live2D 模型。
- 管理员能在后台直观看到每种情绪对应的表情效果和熔岩灯状态。
- 游客端临时调节不会污染后台配置。
- 后台保存的展示参数可以作为新设备默认值。
- 没有引入新的 WebSocket 协议变化。
